2️⃣ API & Database Requirements

A) Withdrawals table (if not already there)

If there is no withdrawals table yet, please create it in Supabase (production DB) with at least:

id (uuid, PK, default gen_random_uuid())

user_id (uuid, FK → users.id)

amount (numeric) — amount user requested

fee (numeric) — always 1

net_amount (numeric) — amount - fee

network (text: 'TRC20' or 'BEP20')

to_address (text) — user’s withdrawal address

status (text: 'pending', 'processing', 'completed', 'failed')

nowpayments_payout_id (text, nullable)

error_message (text, nullable)

created_at, updated_at timestamps


Please also ensure there is a place to store fees (either:

a column like wallets.platform_fee_balance, or

a separate table platform_earnings / earnings_transactions with a record of each 1 USDT fee).


B) New backend route: Create withdrawal request

Create a protected endpoint, something like:

POST /api/wallet/withdraw

Auth required (JWT, current logged-in user)


Request body:

{
  "amount": 120,
  "network": "TRC20",
  "address": "USER_USDT_ADDRESS"
}

Logic:

1. Validate amount >= 10


2. Validate network is "TRC20" or "BEP20"


3. Validate address is not empty


4. Load user wallet from wallets table


5. Check wallet.usdt_balance >= amount


6. Compute:

fee = 1

net_amount = amount - fee



7. If net_amount <= 0, reject


8. Start a transaction:

Insert row into withdrawals with status "processing"

Deduct full amount from wallets.usdt_balance

Add fee to platform fee balance (or insert into platform earnings table)



9. Call a NOWPayments payout API from the backend to send net_amount USDT to user’s address on the selected network.


10. Store nowpayments_payout_id and response data.


11. If payout API call is successful → set status = 'completed'


12. If payout API call fails →

set status = 'failed'

optionally refund the amount: add back (amount) to wallet and reverse fee




Response:

{
  "success": true,
  "message": "Withdrawal created successfully.",
  "withdrawal": {
    "id": "...",
    "amount": 120,
    "fee": 1,
    "net_amount": 119,
    "status": "processing"
  }
}

Or on error:

{ "success": false, "error": "Reason..." }
